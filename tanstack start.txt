=============================================================
UNDERSTANDING TANSTACK AND DEPLOYING STACKWISE
=============================================================

WHAT IS TANSTACK?
-----------------
TanStack is a collection of high-quality, open-source libraries for building modern web applications. The most famous ones are TanStack Query (for data fetching) and TanStack Router (for navigation). 

Recently, they introduced "TanStack Start" (which is what this project uses). TanStack Start is a full-stack React framework built on top of TanStack Router and Vite. It gives you server-side rendering (SSR), file-based routing, and full-stack capabilities, making it a direct competitor to Next.js or Remix.

HOW IT IS USED IN THIS PROJECT
------------------------------
In the Stackwise project, you'll see `@tanstack/react-start` and `@tanstack/react-router` in your package.json. 
- It handles the entire page navigation system (how you move from the Dashboard to the Catalog without the page fully reloading).
- It generates the server/client bundles via Vite when you run `npm run build`.
- It dynamically manages the <head> tags (like the favicon we just updated).

WHY VERCEL AND NETLIFY ARE "REFUSING"
-------------------------------------
The reason your deployments are failing (or showing 404s after deployment) is because of how TanStack Start structures its final build files. 

When you run `npm run build`, a standard React/Vite app puts all the final files in a folder called `dist/`. Vercel and Netlify expect to find an `index.html` file right there in `dist/`.

However, because TanStack Start supports full-stack features, it separates the build into two folders:
1. `dist/client/` (where your actual HTML and assets live)
2. `dist/server/` (where backend code lives)

Because Vercel and Netlify are looking in `dist/`, they find nothing, and thus your deployed site gives a 404 error! Since this project doesn't actually use any complex backend server functions, we can just tell the hosting platforms to serve the `dist/client` folder.

=============================================================
THE BEST PLACE TO DEPLOY (AND STEP-BY-STEP GUIDES)
=============================================================
Both Vercel and Netlify are excellent choices. I highly recommend **Vercel** for this specific stack, as it handles Vite-based projects beautifully.

OPTION A: DEPLOYING TO VERCEL (Recommended)
-------------------------------------------
1. Push your latest code to your GitHub repository.
2. Go to your Vercel Dashboard and click "Add New..." > "Project".
3. Import your Stackwise GitHub repository.
4. In the "Configure Project" screen, look for the "Build and Output Settings" section and click it to expand it.
5. Set "Framework Preset" to: Vite
6. Leave the "Build Command" as default (`npm run build`).
7. **CRITICAL STEP**: Change the "Output Directory" to: `dist/client`
8. Click "Deploy". Your site will now build and the 404 errors will be gone!

OPTION B: DEPLOYING TO NETLIFY
------------------------------
1. Push your latest code to GitHub.
2. Go to your Netlify Dashboard and click "Add new site" > "Import an existing project".
3. Connect your GitHub and select the Stackwise repository.
4. In the "Site settings and deploy" screen, find the "Build settings" section.
5. Set the "Build command" to: `npm run build`
6. **CRITICAL STEP**: Set the "Publish directory" to: `dist/client`
7. Click "Deploy site". 

By explicitly pointing the deployment services to the `dist/client` folder, they will finally find the `index.html` file that TanStack generated, and your application will be perfectly deployable!
